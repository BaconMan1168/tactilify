# Tactilify — Brainstormed Extension Ideas

## 1. SVG Quality Improvements

### Few-shot examples in system prompt (recommended)
Inject 1-2 concrete image→SVG pairs directly into the `/api/llm-tactile` prompt. Claude learns spatial conventions (label placement, shape sizing, line routing) from examples rather than instructions alone. Requires hand-authoring 2-3 good examples (circuit, cell anatomy, chart) as base64 image + SVG pairs. Most effective single prompt technique in the literature (Brown et al. 2020, replicated across all major model families).

### SVG validation + repair loop
After generation, parse the SVG and check key invariants (Braille labels present, viewBox set, no overlapping text, element count reasonable). If it fails, send the SVG back to Claude with the specific error for a focused single repair. Catches the bad ~20% without touching the good 80%. Not a two-pass pipeline — just a post-check retry.

## 3. "Ask About This Diagram" Chat

After the audio walkthrough, student can ask follow-up questions:
- "What does the resistor do in this circuit?"
- "Why is this arrow pointing that way?"
- "Can you explain this more simply?"

**Context:** DiagramAnalysis JSON (already generated by `/api/analyze`) becomes the chat system prompt — no re-processing the image.

**Input modes:**
- Voice: Web Speech API `SpeechRecognition` (browser-native, zero cost)
- Text: standard input as fallback
- Add a new tab for the "chat"
- Make sure that the user can only ask questions related to the diagram only.

**Accessibility:**
- Responses auto-read via Web Speech synthesis (same hook as existing TTS)
- Chat history in `<ul role="log" aria-live="polite">` so screen readers announce new messages
- Each message labeled "You asked:" / "Tactilify says:"
- Voice button keyboard-triggerable (space/enter to record)

**Prompt note:** Claude should answer as if explaining to a blind student — no "as you can see" language. May need to enrich the DiagramAnalysis prompt to include more relational detail (e.g. "resistor connects node A to node B" not just "there is a resistor").

**New route:** `/api/chat` — takes DiagramAnalysis JSON + conversation history + new message → streams response.

## 4. PDF Export for Swell Paper Printing

Swell paper (Repro-Tronics Swell Form, American Thermoform) activates via heat: black ink swells into a raised relief. The tactile SVG already uses black fill/stroke, so it's swell-compatible by design.

**Export feature:**
- Download as PDF — multi-page SVG wrapped into A4 PDF (one SVG per page), correctly sized for standard swell machines
- Print-ready validation — ensure line weights ≥ 1pt and no gray/colored elements that won't activate correctly
- Print instructions panel alongside download: "Print at 100% scale (no scaling) on a laser printer, then run through your swell machine."

No visual swell simulation — misleading since activation depends on machine, paper, and ink density.
